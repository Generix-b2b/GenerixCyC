XML e-Invoicing Standard

Especificación técnica completa del esquema XSD para facturación electrónica B2B en España.

Validaciones

Descripción General

El estándar XML e-Invoicing de Generix Spain proporciona un esquema XSD unificado y modular para la representación de facturas electrónicas en formato XML.
Este estándar está diseñado para cumplir con la normativa española de facturación electrónica y facilitar la integración con sistemas B2B.

Características Principales

  • Compatibilidad: Cumple con los requisitos de la Ley 3/2023 de Facturación Electrónica y normativa fiscal española.
  • Modularidad: Estructura basada en tipos complejos reutilizables que permite componer cada tipo de factura.
  • Validación: El esquema XSD permite validar la integridad estructural y de tipos de cada documento.
  • Escalabilidad: Diseñado para soportar grandes volúmenes de facturas sin degradación de rendimiento.
Nota importante:
Este estándar forma parte del ecosistema de Generix Spain para facturación electrónica.
Se recomienda revisar la documentación completa antes de implementar.

Reglas de Validación

Validaciones Estructurales

Estructura XML

El XML debe ser válido y estar bien formado, y validar correctamente contra el esquema XSD.
Se rechazarán documentos con errores de sintaxis.

Campos Requeridos

Todos los elementos sin minOccurs="0" son obligatorios y deben estar presentes y no vacíos.

Tipos de Datos

Cada elemento debe cumplir con el tipo XSD especificado (xs:string, xs:integer, xs:decimal, tipos con patrón, Enums).
