GECOM10 · Supplier Master Data

Especificación técnica del esquema JSON para la carga de datos maestros de proveedores (Supplier Master Data) en el WMS de Generix.

Descripción General

El esquema GECOM10 — Supplier Master Data define la estructura JSON para el intercambio de datos maestros de proveedores con el sistema de gestión de almacén (WMS) de Generix.
Cada fichero contiene una colección de proveedores, con sus datos de identificación, dirección, contacto, parámetros logísticos y dos bloques de declaración adicional (GEEX1001 y GEEX1002).

Características Principales

  • Colección de proveedores: el elemento raíz supplierMasterData es un array que admite uno o varios proveedores en el mismo fichero.
  • Códigos de operación: el campo TRTEXC indica la acción a realizar (alta, modificación, baja…).
  • Datos extendidos: los bloques GEEX1001 y GEEX1002 agrupan información adicional opcional del proveedor.
  • Listas cerradas: varios campos se validan contra listas de valores admitidos (ver la pestaña Enumeraciones).

¿Qué representan los bloques GEEX1001 y GEEX1002?

GEEX1001 es la Supplier Additional 01 declaration (información adicional: responsable, email, códigos logísticos…) y GEEX1002 es la Supplier Additional 02 declaration (comentarios y nombres adicionales del proveedor). Ambos bloques son opcionales.

¿Cómo se codifica el país del proveedor?

El campo PAYFOU usa códigos de país ISO 3166-1 (alpha-2 o alpha-3), por ejemplo ES / ESP para España.

¿Para qué sirve el campo TRTEXC?

Indica el tipo de movimiento sobre el registro del proveedor: 1 = alta, 2 = modificación o alta, 7 = modificación de cabecera, 9 = cancelación.

¿Puedo enviar varios proveedores en un mismo fichero?

Sí. supplierMasterData es un array y admite varios objetos proveedor en la misma carga.
