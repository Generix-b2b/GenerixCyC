JSON e-Invoicing Standard

Especificación técnica completa de la estructura JSON para facturación electrónica B2B en España.

Descripción General

El estándar JSON e-Invoicing de Generix Spain proporciona una estructura unificada y modular para la representación de facturas electrónicas en formato JSON. Este estándar está diseñado para cumplir con la normativa española de facturación electrónica y facilitar la integración con sistemas B2B.

Características Principales

  • Compatibilidad: Cumple con los requisitos de la Ley 3/2023 de Facturación Electrónica y normativa fiscal española.
  • Modularidad: Estructura flexible que permite incluir solo los campos necesarios para cada tipo de factura.
  • Validación: Incluye reglas de validación integradas para garantizar la integridad de los datos.
  • Escalabilidad: Diseñado para soportar grandes volúmenes de facturas sin degradación de rendimiento.
  • Interoperabilidad: Compatible con sistemas EDI, APIs REST y otros estándares de intercambio de datos.
Nota importante: Este estándar forma parte del ecosistema de Generix Spain para facturación electrónica. Se recomienda revisar la documentación completa antes de implementar.

Versiones Soportadas

  • Versión 2.0 - Versión actual (recomendada)
  • Versión 1.5 - Mantenimiento hasta 2025
  • Versión 1.0 - Deprecada (no se recomienda su uso)

Reglas de Validación

Validaciones Estructurales

Estructura JSON

El JSON debe ser válido y bien formado.
Se rechazarán documentos con errores de sintaxis.

Campos Requeridos

Todos los campos marcados como obligatorios deben estar presentes y no vacíos.

Tipos de Datos

Cada campo debe cumplir con el tipo especificado (String, Integer, Decimal, Date, Object, Array, Enums).

Validaciones de Negocio

NIF/CIF

  • Formato: 8 dígitos + 1 letra (ej: A12345678)
  • La letra debe corresponder al código de validación NIF
  • CIF válidos también aceptados (B87654321)

Fechas

  • Formato requerido: YYYY-MM-DD
  • La fecha de emisión no puede ser futura
  • La fecha de vencimiento debe ser posterior a la de emisión

Importes

  • Cantidad y precioUnitario deben ser positivos
  • Máximo 2 decimales en importes
  • El importe de línea = cantidad × precioUnitario - descuento
  • Total debe coincidir con suma de líneas + IVA - retenciones

Respuestas de Validación

Preguntas Frecuentes

¿Cuál es la diferencia entre factura y factura rectificativa?

Una factura rectificativa se utiliza para corregir errores en una factura ya emitida. Debe incluir referencia a la factura original y el motivo de la rectificación. A nivel fiscal es una factura independiente que se emite junto a la original.

¿Cómo manejar descuentos?

Los descuentos se aplican a nivel de línea (campo descuento) o pueden agregarse a nivel de factura (campo descuentoGlobal en totales). El importe final de línea se calcula como: (cantidad × precioUnitario) - descuento.

¿Se pueden incluir retenciones?

Sí, a través del campo retencion en el objeto totales. Las retenciones se aplican al importeTotal y generalmente son del 19% para prestación de servicios en actividades profesionales.

¿Qué hago si mi empresa tiene múltiples sedes?

Use la dirección fiscal oficial de la empresa en el campo emisor.direccion. Si necesita incluir una dirección diferente para envío, puede agregarse en un campo adicional direccionEnvio.

¿Cuál es el máximo de líneas que puede tener una factura?

No hay límite técnico establecido, pero se recomienda mantener facturas manejables. Para volúmenes muy grandes, considere fragmentar en múltiples facturas.

¿Cómo integro esto con mi sistema actual?

Generix proporciona:

  • APIs REST para validación y transformación
  • Librerías client para JavaScript, Python y Java
  • Conectores directos para ERP populares (SAP, Dynamics, etc.)

Consulte con nuestro equipo de soporte técnico para opciones específicas a su tecnología.
